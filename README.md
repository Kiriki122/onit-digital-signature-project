# Защищенная передача файлов по Email (ЭЦП + Шифрование)

Данное консольное приложение реализует защищенный документооборот между двумя сторонами (Отправителем и Получателем) с использованием электронной почты.

Приложение обеспечивает:

- **Конфиденциальность:** файл шифруется с помощью гибридного шифрования (RSA 2048 + AES/Fernet).
- **Целостность и неотрекаемость (ЭЦП):** файл подписывается закрытым ключом отправителя (RSA + SHA-256).
- **Автоматизацию:** отправка по протоколу SMTP и чтение почты по протоколу IMAP.

## 📋 Требования (Prerequisites)

Перед началом работы убедитесь, что у вас есть:

1. Установленный **Python 3.8** или выше.
2. Два почтовых ящика (можно использовать Gmail, Yandex, Mail.ru).
3. **Пароли приложений** для каждого ящика.
    > _Внимание: обычные пароли от почты не подойдут. Вам нужно зайти в настройки безопасности вашего почтового провайдера и выпустить "Пароль приложения" (App Password)._

## 🛠 Установка и настройка

**1. Клонирование / подготовка проекта**
Откройте терминал в папке с проектом и создайте виртуальное окружение:

```bash
python -m venv venv
```

Активируйте виртуальное окружение:

- Для Windows: `venv\Scripts\activate`
- Для Linux/Mac: `source venv/bin/activate`

Установите необходимые зависимости:

```bash
pip install -r requirements.txt
```

**2. Настройка переменных окружения**
В корне проекта находится файл `.env.example`. Скопируйте его и переименуйте в `.env`:

```bash
cp .env.example .env  # или просто переименуйте файл вручную
```

Откройте файл `.env` и заполните его своими реальными данными:

```env
# Настройки для отправки (Отправитель)
SMTP_SERVER=smtp.gmail.com # Возможные варианты: smtp.yandex.ru, smtp.mail.ru
SMTP_PORT=465
SENDER_EMAIL=your_sender_email@gmail.com
SENDER_PASSWORD=your_sender_app_password

# Настройки для приема (Получатель)
IMAP_SERVER=imap.mail.ru # Возможные варианты: imap.gmail.com, imap.yandex.ru
RECEIVER_EMAIL=your_receiver_email@mail.ru
RECEIVER_PASSWORD=your_receiver_app_password
```

## 🚀 Как использовать (Запуск)

Работа с приложением состоит из трех шагов. Команды необходимо выполнять в активированном виртуальном окружении.

### Шаг 1: Генерация ключей

Перед первой отправкой нужно сгенерировать пары асимметричных ключей (RSA) для отправителя и получателя. Выполните:

```bash
python generate_keys.py
```

_Результат:_ В папке появятся 4 файла `.pem` (публичные и приватные ключи для обеих сторон).

### Шаг 2: Подписание, шифрование и отправка

Запустите скрипт отправителя. Он создаст тестовый секретный файл, подпишет его хэш, зашифрует данные и отправит на почту получателю:

```bash
python sender.py
```

_Результат:_ В консоли появится сообщение об успешной отправке.

### Шаг 3: Прием, расшифровка и проверка ЭЦП

Подождите несколько секунд, чтобы письмо дошло до почтового сервера, и запустите скрипт получателя:

```bash
python receiver.py
```

_Результат:_ Скрипт подключится к ящику получателя, найдет нужное письмо, скачает зашифрованные вложения, расшифрует их, проверит ЭЦП и сохранит исходный файл в папку проекта с префиксом `decrypted_`.

## 📁 Структура проекта

- `generate_keys.py` — утилита для генерации RSA-ключей.
- `sender.py` — логика создания ЭЦП, гибридного шифрования и отправки письма (SMTP).
- `receiver.py` — логика поиска письма (IMAP), расшифровки и валидации подписи.
- `.env.example` — шаблон файла конфигурации.
- `*.pem` — файлы ключей (генерируются автоматически).
